# [WIP-draft] 비중 슬라이드 설계 프레임워크 v1

> ⚠ **WIP — 야간 자율 draft. 미승인.** 구현·commit·채택 금지. 아침(07:00 KST) 이사님 승인 후 백테스트 단계로.
> 발신: `@heav_lnx_trader_bot` / 배정 헤더 "야간 자율 배정 (01:00 KST, 07:00 취합)".

## 1. 배경 — exit_ratio 리서치 결론을 어떻게 받을 것인가

`research_exit_ratio.py` 결과(work-queue §autotrader 요약):
- **exit=0(매도X)** 가 전 기간·심볼에서 수익 압도 — QQQ 2007-26 **+1271%** vs exit=1 **+119%**.
- exit↑(매도 비중↑) 일수록 수익 ↓, MDD는 ↓(방어 우수), Sharpe는 exit=0이 최고.
- **원인:** BEAR 매도→재진입 타이밍 베팅이 **상승장 복리 폭증**을 훼손. 라오어 본질 엣지(하락매수·평단가하락)를 매도가 깸.

**딜레마:** 수익은 exit=0(매도 금지), 방어는 exit=1(적극 매도). 양극단 사이에서 **"매도 없이 MDD만 부분 제어"** 가 비중 슬라이드의 목표.

## 2. 설계 원칙 (★ 핵심 통찰)

> **비중 슬라이드 = "매도 타이밍 베팅"이 아니라 "자본 배분의 사전 관리".**

1. **코어 보유물은 절대 매도 안 함** (exit=0 교훈). 복리 폭증 엔진 보존.
2. 매도 대신, **매월 투입 예산(`base`)을 주식/현금으로 분할** — 가격 밴드에 따라 슬라이드.
3. **고가에서 덜 사** 현금(탄약)을 쌓고, **저가에서 더 사** 그 탄약을 투입. (라오어 하락가산의 확장형)
4. exit_ratio의 BEAR 매도는 제거. BEAR에서도 매도 않고, 적립 배분만 소극 주식화.

→ exit=0의 "복리 엔진"은 살리고, "항상 풀매수"로 인한 고점 노출만 줄인다.

## 3. 가격 밴드 정의 (sma200 기준, 기존 d 재사용)

`d = (sma200 - price) / sma200` (양수=가격이 기준선 대비 하락). 기존 `_down_budget`와 동일 변수.

| 밴드 | 조건(d) | 상태 |
|---|---|---|
| B1 과열 | d < -0.05 | 상단 돌파 |
| B2 보통상승 | -0.05 ≤ d < 0 | 기준선 부근(상승) |
| B3 약세 | 0 ≤ d < 0.10 | 기준선 하회 |
| B4 깊은하락 | 0.10 ≤ d < 0.25 | 조정 |
| B5 침하 | d ≥ 0.25 | 하락장/위기 |

## 4. 밴드별 월 예산(`base`) 배분 매핑표 (initial, 검증 대상)

| 밴드 | 주식 매수 비율 `w(d)` | 현금(탄약) 비율 | 탄약 투입 `deploy(d)` |
|---|---|---|---|
| B1 과열 | 30% | 70% (축척) | 0% |
| B2 보통상승 | 50% | 50% | 0% |
| B3 약세 | 80% | 20% | 25% |
| B4 깊은하락 | 100% | 0% | 75% |
| B5 침하 | 100% | 0% | 100% (+ cap) |

- `w(d)` = 해당 월 `base` 중 주식으로 갈 비율. 잔여 = 현금 탄약 누적.
- `deploy(d)` = 누적된 현금 탄약 중 이번 달 추가 매수에 풀 비율. 저가 밴드일수록 탄약까지 동원.

## 5. 평단가 하락 정량화 모델

```
# 기존(라오어, exit=0):
budget_old = min(base * (1 + k*max(0,d)), base*cap)        # 항상 주식 매수

# 비중 슬라이드(본안):
budget_stocks = base * w(d)                                 # 배분분
budget_stocks += accumulated_cash_reserve * deploy(d)       # 저가 시 탄약 투입
budget_stocks *= (1 + k * max(0, d))                        # 라오어 하락가산 유지(저가에서 증량)
budget_stocks = min(budget_stocks, base * cap)              # cap 상한(비용 폭발 방지)
# 단, B4/B5에서 cap 한도를 넓히는 변형(initial: cap*1.5) 검증 대상 — 탄약 투입 허용치
```

**평단가 효과(추정):**
- 과열(B1)·보통상승(B2)에서 `base`의 50~70%가 현금으로 빠짐 → 비싼 가격에 적게 사 → **평단가 상승 억제**.
- 약세 이하(B3~B5)에서 탄약까지 가산 → **평단가 하락 강화**(라오어보다 더 낮은 평단 가능).
- 정합: 월 적립 예산 총액은 보존(주식+현금=base+탄약), 매도로 자본이 빠져나가지 않음.

## 6. 예상 메트릭 추정 (백테스트 전 — 합리적 추정, ★ 검증 대상)

exit=0 / 본안 / exit=1 의 3점 사이에서 본안은 exit=0에 가깝되 MDD를 끌어내리는 방향:

| 지표 | exit=0 (기존) | **비중 슬라이드(본안 추정)** | exit=1 (방어 극대) |
|---|---|---|---|
| 총수익률(QQQ 07-26 추정) | +1271% | **+900~1150%** (고점 비중 축소로 일부 희생) | +119% |
| MDD | 가장 큼(exit↑록 개선) | **exit=0 대비 유의 개선** (현금 완충) | 가장 작음 |
| Sharpe | (이미 최고급) | **exit=0 근접~소폭 상회** (변동성↓ 효과) | 중간 |
| 평단가 | 기준 | **더 낮음**(과열 억제+저가 가산) | 높음(재매수 왜곡) |

- 수익은 exit=0보다 **소폭 낮아도** MDD·Sharpe에서 보상 → 위험조정수익 우위 가능.
- "exit=0의 수익을 약간 희생하더라도 하락장 완충을 산다"가 본안의 베팅.

## 7. MDD 제어 한계 (솔직한 인식)

- 코어 보유물을 매도하지 않으므로, **극단적·지속 하락(2008/2022급)에서는 현금 비중만으로 MDD 제어에 한계**.
- exit=1 수준의 방어는 불가. 본안의 목표는 "exit=0의 수익 곡선을 누르지 않으면서 낙폭만 다듬는 것" — 방어의 극대화가 아님.
- 이 한계가 수용 불가면 → (c)재진입 개선 또는 exit_ratio 소수치(0.1~0.2) 병용 검토.

## 8. 통찰 / 후보 방향 (영감)

1. **"매도 결정 = 타이밍 예측"** 인 반면, **"배분 결정 = 상태 함수"** — 예측 없이 가격 밴드(현재 상태)만 본다. 라오어 철학(예측 배제)과 일관.
2. 현금 탄약 = **옵션성(optionality)**. 저가 때 쓸 수 있는 권리를 비싼 가격에 미리 비축.
3. 밴드 경계(예: d=-0.05) 근처의 **갑작스런 전환** 효과 → 부드러운 슬라이드(선형 보간) vs 계단형 토글, 둘 다 백테스트 비교 필요.
4. 파생: `w(d)`를 밴드 고정표 대신 **연속함수**(예: `w(d)=clip(0.3+1.4d, 0.3, 1.0)`)로 매핑하는 변형 — 검증 대상.
5. **레짔 필터(BULL/BEAR 모멘텀)와의 결합:** 본안은 exit_ratio(매도)를 대체하지만, 레짔의 "BEAR 시 매수 중단"은 유지 가능 — 매도는 제거, 매수 강도만 슬라이드. 이 조합이 본안의 완전형.

## 9. 검증 계획 (아침 승인 후)

1. `WeightSlideStrategy` 클래스 구현(인터페이스 동일) — `on_rebalance`에서 `w(d)·deploy(d)` 적용.
2. 동일 데이터·기간(QQQ/SPY, 2007-26/2020/2010)에서 exit∈{0,0.3,0.5,0.7,1.0} + 본안 비교.
3. 메트릭: 수익률·MDD·Sharpe·평단가 궤적·월별 승률.
4. **검증 루프(사칙):** 서로 다른 관점 서브에이전트(수익 관점 / 방어 관점) 병렬 비판 → Codex 재검증 최소 2회.
5. 백분위·워크포워드는 v1 검증 후(전략 스펙 §6 리스크 인프라와 함께).

## 10. 미결정(승인 안건)

- `w(d)`·`deploy(d)` 수치(initial 위 표) — 도메인 지식 기반 초기값, **반드시 백테스트로 교정**.
- B4/B5에서 cap 확대 여부(cap*1.5).
- 밴드 계단형 vs 연속형 매핑.
- 레짔 매수중단과의 결합 형태.
