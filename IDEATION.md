> **상태:** 초안 (IDEATION) · 미인증 산출물 · **미확정 스택** → 발판/커밋/ADR 전 금지
> **날짜:** 2026-06-26
> **작성:** trader 팀장 + 직원(subagent) 4명 병렬 조사
> **원칙:** 사용자 아이디어·취향 추측/반영 없이 **독자 아이디에이션**. 뻔한 평균수렴(RSI/볼린저 단타, 단순 그리드, BTC 현물 모멘텀, 순방향 SMA)은 배제.

---

## 1. 핵심 가설 — 잔존 엣지는 "구조적 단편화/단절"에서 나온다

2026년 시점, **단순 알파**(BTC/ETH 현물 차익, 미국 대형주 모멘텀, 이더리움 메인닷 DEX 아비트라지)는 2024–25년 이미 평균수렴 — 신규 진입자 마진 거의 없음. 반면 **시장을 단절하는 구조적 요인**(규제·국경·자산군 경계·오라클 지연·거래소 독점·청산 메커니즘)이 존재하는 한 알파도 지속된다.

> 설계 철학: 시장 방향을 맞히려(예측) 하기보다, **구조가 만들어내는 스프레드/왜곡을 포착**하는 쪽이 (a) 소규모 자본에 열려 있고, (b) 알파 소진이 느리고, (c) 통계적으로 검증하기 쉽다. 예측은 부차.

이 가설을 4축(시장·전략·인프라·리스크)에서 전개한다.

---

## 2. 시장·자산 — 어디에 엣지가 남아있나

뻔한 곳(현물 단타, 대형주) 배제 후 잔존 엣지 후보:

| 방향 | 구조적 단절의 원인 | 접근성 | 비고 |
|---|---|---|---|
| **거래소 간 펀딩/베이스 분산** | 담보 이동 지연·사용자 베이스 차이 | 중(CEX API) | 단순 cash-and-carry는 포화, but 스프레드 분산/basis 비대칭은 잔존 |
| **예측시장 ↔ 현물 (Kalshi perps / Polymarket)** | 신상품(2026 런칭) 골든타임, Chainlink 오라클 2–15초 선행창 | 중~고 | 확률합>100% 자동 탐지; 지능 장벽으로 포화 지연 |
| **국경/KRW 상장 프리미엄 (Upbit 신규 상장, 이머징 P2P NGN/ARS/TRY)** | 자본통제·거래소 독점 | 중~고(현물+P2P) | 헤지 다리 결합 시 방향중립적 프리미엄 수확 |
| **크립토 옵션 (Deribit)** | IV>RV 지속(보험수요), 덜 효율적 | 중 | 변동성 프리미엄·디스퍼전 |

**배제 근거:** MEV/DEX 순수 아비트라지는 90%+가 프라이빗 채널 라우팅 → 소자본·개인 진입 사실상 불가.

## 3. 전략 클래스 — 덜 붐비는 정량 카테고리

8개 카테고리 평가 후, 소규모 자본·구현가능성·잔존 알파 기준 상위 3:

1. **온체인/대체데이터 정성→정량** — 고래 흐름·NVT·센티먼트·개발자 활동을 LLM/ML로 정량 신호화. **잔존 알파 최대**. 차별화 = 데이터 인프라+신호화 파이프라인. 방향성(변동 큼).
2. **통계차익 — 코인테그레이션/코플라 페어** — 롤링 cointegration + 동적 헤지. 시장중립. 핵심 과제 = 레짐 전환 시 관계 붕괴 관리.
3. **변동성·변동성패턴 거래** — Deribit 옵션 기반 VRP/볼 스프레드 + 디스퍼전. 관건 = 숏 볼 테일 리스크 통제.

> 공통: 중빈도(분~시간), 방향중립 또는 헤지된 방향성, 구조적 메커니즘에 기반(예측 아님).

---

## 4. 실행·데이터 인프라 — 백테스트↔라이브 parity가 1등 기준

- **parity 우선:** 같은 코드가 백테스트·라이브 양쪽에서 동작해야. **NautilusTrader**(Rust 코어 + Python API)가 parity 설계 목표상 사실상 1위. vectorbt = 탐색 전용, freqtrade = 단순 전략용.
- **레이턴시 현실:** 차익 손익분기 5–25ms. 실측 tick-to-trade Python ~12ms(스파이크 80ms) vs Rust ~40µs(~300x). 소자본·싱글서버 = **중빈도 전략만 가능**; HFT/MM은 인프라 비용이 자본을 초과 → 배제.
- **데이터:** 거래소 **무료 덤프 → Parquet/DuckDB**가 80% 커버(비용 0). 정밀 L2 검증 단계에만 Tardis 크레딧.
- **호스팅:** 단일 EC2로 시작; 베어메탈이 AWS 대비 50–80% 절감.

---

## 5. 리스크·검증 — 백테스트는 "나쁜 전략 거르기"용

- 백테스트는 좋은 전략을 **증명**하는 게 아니라 나쁜 전략을 **거르는** 용도. 시도 횟수 미보고 시 사기에 가까움(López de Prado).
- **과적합 방어:** 시도 카운터 로깅 + **Deflated Sharpe Ratio(DSR)** + 사전 가설 고정. Harvey(2016): 새 팩터는 t≥3.0. 90%+ 학술 전략이 라이브 실패.
- **포지션 사이징:** 부분 Kelly(¼~½) — 파국위험·엣지 과대추정·하방 백분위 최적화가 중첩(overdetermined).
- **페이퍼→라이브 갭:** 슬리피지·지연·용량·숨은 유동성이 주원인 → 이 갭을 **백테스트 안에서 비관적으로 모델링**(동적 슬리피지·시장충격·용량 한도).
- **킬스위치:** 위반 "직전"에 발동. 3단계 회로차단기(경고→축소→전면중단) + 강제 쿨다운 ≥2주 + **인간 승인 재기동**.
- **검증 의무:** 단일 백테스트 결과 보고 금지. Walk-Forward(≥30폴드) + Monte Carlo(5종 ≥500회) + PBO/DSR. 산출물 = 숫자가 아니라 **성과 분포와 5백분위 콘**. 라이브가 콘 이탈 시 "불운"이 아닌 "깨진 전략"으로 자동 중단.

## 6. 통합 — 독창 방향 시나리오 3개 (후보)

조사를 교차하면 뚜렷한 3개의 방향이 떠오른다. 모두 크립토 파생상품/온체인 중심·중빈도·소자본 가능·구조적 엣지.

### 시나리오 A — **구조적 단절 차익 엔진** (방향중립)
여러 구조적 단절(CEX↔DEX, 거래소간 펀딩, 국경/P2P, 예측시장↔현물)이 만드는 스프레드를 **하나의 이벤트 기반 엔진**로 포착. 베이스라인 = 거래소간 펀딩/베이스 carry; 확장 = Kalshi/Polymarket 오라클 선행창, Upbit KRW 상장 과대반응. 방향중립·자본효율 좋음·검증 쉬움. 가장 "뻔하지 않은" 통합 형태.

### 시나리오 B — **온체인 정성→정량 신호 + CEX 실행** (방향성)
LLM/ML이 온체인+대체데이터(고래·NVT·센티먼트·개발자 활동)를 정량 신호로 변환 → CEX 중빈도 포지션. 잔존 알파 최대로 평가됨. 차별화는 데이터·신호화 인프라. LLM 시대의 새로운 엣지 형태. 인프라 집중도 높음.

### 시나리오 C — **크립토 변동성 프리미엄 수확** (변동성/옵션)
Deribit 옵션 기반 VRP·볼 스프레드·디스퍼전. 크립토 옵션은 아직 덜 효율적(IV>RV 지속). 숏 볼 테일 통제가 관건. 구조적(보험수요 프리미엄)이라 지속 가능.

> 세 시나리오는 **상호 배타가 아님** — 공통 백본(이벤트 엔진 + parity 백테스트 + 회로차단기) 위에 전략 모듈로 올릴 수 있다.

## 7. 미결정 사항 (사용자 결정 필요 → ADR 트리거)

아래 축을 사용자가 선택하면 스택·엔진·시장·전략 ADR로 발전시킨다.

1. **자본 규모 & 레버리지 허용도** — 소자본 중빈도 vs 확장 지향
2. **전략 방향** — 방향중립 차익(A) vs 방향성 신호(B) vs 변동성(C)
3. **시장 범위** — 크립토 단일 vs 전통자산/혼합 확장
4. **실행 단계** — 라이브 거래 의향 vs 연구/페이퍼 우선
5. **시간 지평** — 데이(중빈도) vs 스윙/포지션
